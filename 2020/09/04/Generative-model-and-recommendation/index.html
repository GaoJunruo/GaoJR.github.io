Generative model and recommendation

生成模型在推荐系统中的应用

1. Generative Adversarial User Model for Reinforcement Learning Based Recommendation System（ICML 2019 ）

a. 提出的问题：

用户的interest会随着她观察到的事物而改变，推荐系统可以通过改变展示给用户item隐藏另一部分item来指导用户的interest。本文要获得用户的长期interest。

b. 采用的方法：

主体是RL；要通过生成对抗网络模拟用户动态行为并学习奖赏函数；提出Cascading DQN算法用来获得一系列候选item

c. 存在的问题(困难)：

  1. 用户的interest如何指导用户的行为是未知的，但是却影响着RL算法，遗忘的方法都+1-1代表click/no-click，但是不能表达用户对不同item的偏好。
  2. 无模型 RL 一般都需要和环境（在线用户）进行大量的交互才能学到良好的策略。但这在推荐系统设置中是不切实际的。为了解决无模型方法样本复杂度大的问题，基于模型的 RL 方法更为可取。

    d. 解决方案：

    生成模型模拟用户的行为，利用rl来反馈奖励。

    3. LARA: Attribute-to-feature Adversarial Learning for New-item Recommendation（wsdm2020）

    a. 提出问题：(生成模型解决冷启动问题)

    推荐系统中存在冷启动的问题，从item的属性不同侧面生成用户表示。因此基于属性层的相似度，可以将新的item推荐给用户.

    b. related work：

    以往生成user-item交互的方法：IRGAN，GraphGAN，CFGAN。向量的每个维度都反映了用户购买相应商品的可能性。因此，所生成的交互向量可以被视为相应用户的表示。（Each dimension of the vector reflects the probability that the user will buy the corresponding item. Thus, the generated interaction vectors could be regard as the representation of the corresponding users. ）

    c. 解决方案：

    生成模型是用多个生成器生成和item属性相关的user profiles。判别器是用来判别生成的profiles和真正的profiles。核心就是GAN生成用户表示

4.CFGAN: A Generic Collaborative Filtering Framework based on Generative Adversarial Networks

a. 传统GAN应用在推荐系统中存在的问题（IRGAN和GraphGAN）：

在判别器的“指导”下，随着训练的进行，生成器G将生成与真实情况完全相同的项目ID。但是，这对判别器来说并不友好，同一个物品，它有可能既被标记为真实数据（real），又被标记为生成数据（fake）

b. 解决方法：

提出一种vector-wise的训练方法。生成器生成的向量是购买向量（类似于隐式反馈的user-item矩阵）。向量中每一个值都代表用户与商品的交互概率。
